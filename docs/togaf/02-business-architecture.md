## 1. 文档概述

### 1.1 目的
本文档定义 EHS 智能安保决策中台的业务架构，描述业务流程、组织结构和业务能力。

### 1.2 范围
- **业务流程**: 告警处理、预案检索、事故分析、系统管理
- **组织结构**: 用户角色、权限体系
- **业务能力**: 核心业务服务、支撑服务

## 3. 业务流程

### 3.1 核心业务流程：告警处理
```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ 传感器  │────▶│ 告警上报 │────▶│ 风险分级 │────▶│ 预案检索 │────▶│ 处置执行 │
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘
     │               │               │               │               │
     │               ▼               ▼               ▼               ▼
     │        记录告警信息     调用风险模型      GraphRAG 检索    执行预案步骤
     │        - 时间地点       - LLM 分析         - 相似度匹配      - 处置指南
     │        - 告警类型       - 规则引擎         - 三路召回        - 人员调度
     │        - 传感器数据     - 分级输出         - 重排序          - 结果记录
     │
     ▼
┌─────────┐
│ 事后分析 │
│ - 根因分析
│ - 改进建议
│ - 报告生成
└─────────┘
```

## 6. 业务场景

### 6.1 场景 1: 气体泄漏告警处置
```
1. 传感器检测到气体浓度超标
2. 自动上报告警到 EHS 中台
3. 风险分级模型评估为"高风险"
4. GraphRAG 检索气体泄漏预案
5. 推送预案到安保人员手持终端
6. 安保人员执行处置步骤
7. 处置完成，系统记录归档
8. EHS 工程师生成事故报告
```

### 6.2 场景 2: 复杂事故分析
```
1. 多传感器同时告警 (温度 + 气体 + 烟雾)
2. 系统聚合关联告警
3. 调用 LLM 进行根因分析
4. 生成事故时间线
5. 推荐处置优先级
6. 管理人员审批处置方案
7. 跨部门协同处置
8. 事后复盘和改进建议
```

### 6.3 场景 3: 预案管理与更新
```
1. EHS 工程师创建新预案
2. 系统验证预案格式
3. 提交管理人员审批
4. 审批通过后发布
5. 预案版本归档
6. 通知相关人员
7. 旧预案保留可追溯
```

## 8. 业务指标 (KPI)

### 8.1 运营指标
| 指标 | 定义 | 目标值 | 测量方式 |
|------|------|--------|----------|
| 告警响应时间 | 告警发生到首次响应 | < 8min | 系统自动记录 |
| 处置完成率 | 按时完成处置的告警比例 | > 95% | 处置记录统计 |
| 预案检索准确率 | 检索结果相关性 | > 90% | 用户反馈 |
| 系统可用性 | 系统正常运行时间 | > 99.9% | 监控统计 |

### 8.2 质量指标
| 指标 | 定义 | 目标值 | 测量方式 |
|------|------|--------|----------|
| 风险分级准确率 | 分级正确的告警比例 | > 95% | 人工复核 |
| 误报率 | 错误告警占总告警比例 | < 5% | 统计计算 |
| 漏报率 | 未检测到的事故比例 | < 1% | 事后分析 |
