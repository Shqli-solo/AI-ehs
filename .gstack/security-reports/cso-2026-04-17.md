## CRITICAL 级别发现

### C-1: JWT Secret 默认值可预测（跨两服务）

**影响**: 攻击者可伪造任意用户 JWT token，完全绕过认证

**位置**:
- `apps/ehs-ai/src/adapters/primary/auth.py:13` — `JWT_SECRET = os.getenv("JWT_SECRET", settings.APP_NAME + "_default_secret_change_in_prod")`
- `apps/ehs-business/src/main/resources/application.yml:35` — `jwt.secret: ehs-jwt-secret-key-2026-production-change-me`

**攻击场景**:
1. 生产环境部署时，若未正确设置 `JWT_SECRET` 环境变量，Python AI 服务默认使用 `EHS AI Service_default_secret_change_in_prod`
2. Java 服务默认使用 `ehs-jwt-secret-key-2026-production-change-me`
3. 攻击者知道默认密钥后，可生成任意用户的 JWT token
4. 两服务使用不同的默认密钥，若环境变量只配了一个，另一服务会回退到默认值，导致认证不一致

**修复**:
```python
# auth.py - 生产环境必须有密钥，不允许默认值
JWT_SECRET = os.environ["JWT_SECRET"]  # 不设默认值，缺失时直接崩溃
```
```yaml
# application.yml - 移除硬编码值
jwt:
  secret: ${JWT_SECRET:}  # 空值时启动失败
```

### C-2: 无角色/权限控制 — 任何认证用户拥有全部访问权限

**影响**: 水平越权，普通用户可访问管理员功能

**位置**: `apps/ehs-business/src/main/java/com/ehs/business/config/SecurityConfig.java`

**证据**: 全代码库搜索 `@PreAuthorize`, `@Secured`, `@RolesAllowed` — 零结果。SecurityConfig 仅区分 `permitAll()` 和 `authenticated()`，没有任何角色或权限检查。

**攻击场景**:
1. 注册普通用户账号（如"值班经理"角色）
2. 使用该用户 JWT 访问所有 API 端点
3. 可删除用户、修改配置、触发告警 — 无任何权限拦截

**修复**:
```java
// SecurityConfig.java - 添加角色区分
.requestMatchers("/api/admin/**").hasRole("ADMIN")
.requestMatchers("/api/users/**").hasAnyRole("ADMIN", "MANAGER")
.anyRequest().authenticated()

// 在 Controller 方法上
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<Void> deleteUser(@PathVariable Long id) { ... }
```

## HIGH 级别发现

### H-1: Java 服务 Actuator 端点公开暴露

**影响**: 泄露系统内部信息，包括环境变量、Bean 配置、线程 dump

**位置**: `apps/ehs-business/src/main/resources/application.yml:56-63`

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
```

在 SecurityConfig 中 `/actuator/**` 被设为 `permitAll()`（line 68），意味着任何人都可以访问 actuator 端点。

**攻击场景**:
1. 访问 `/actuator/env` 获取所有环境变量（包括数据库连接串）
2. 访问 `/actuator/health` 获取系统状态
3. 结合其他漏洞，可定位内部服务拓扑

**修复**: 从 permitAll 白名单中移除 `/actuator/**`，或限制仅内网访问：
```java
.requestMatchers("/actuator/**").hasRole("ADMIN")
```

---

### H-2: Python AI 服务以 root 用户运行

**影响**: 容器逃逸后可获得宿主机 root 权限

**位置**: `apps/ehs-ai/Dockerfile` — 缺少 `USER` 指令

**对比**: admin-console 和 ehs-business 的 Dockerfile 都有非 root 用户配置：
```dockerfile
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser
```
ehs-ai 的 Dockerfile 缺少此配置，容器以 root 运行。

**修复**:
```dockerfile
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser
```

## MEDIUM 级别发现

### M-1: 文件上传未验证文件名（潜在路径穿越）

**影响**: 攻击者可上传恶意文件名覆盖存储桶对象

**位置**: `apps/ehs-ai/src/adapters/primary/rest_api.py:492`
```python
object_name=f"alerts/{alert_id}/{image.filename}",
```

`image.filename` 直接使用，未做 sanitization。攻击者可上传 `../../../etc/passwd` 类型文件名。

**修复**: 使用 UUID 或 hash 重命名上传文件，不信任用户提供的文件名。

---

### M-2: Prompt Injection 风险 — 用户输入直接拼入 LLM 提示词

**影响**: 攻击者通过精心构造的告警消息操控 LLM 行为

**位置**: `apps/ehs-ai/src/core/agents/risk_agent.py:79-81`
```python
def _build_prompt(self, alert_message: str) -> str:
    return f"""请分析以下告警信息...
告警信息：{alert_message}
```

用户提供的 `alert_message` 未经任何过滤直接插入系统提示词。

**修复**: 使用 LangChain 的 ChatPromptTemplate 分离系统消息和用户消息，添加 prompt injection 防护层。

---

### M-3: 前端依赖漏洞 — Next.js 14.2.0 存在已知 CVE

**影响**: GHSA-3h52-269p-cp9r — prefetch 请求缓存中毒

**证据**: `pnpm audit` 显示 23 个漏洞（1 critical, 2 moderate, 20 low）

**修复**: 升级 Next.js 到最新版本 `pnpm add next@latest`

---

### M-4: 无 API 速率限制（应用层）

**影响**: 暴力破解、DoS 攻击

**证据**: 全代码库搜索 `rate.limit`, `throttle` — 零结果。K8s Ingress 层有 rate-limit annotations（100 req/min），但应用层无防护。

**修复**: 在 FastAPI 中添加 slowapi 或自定义 rate limiter；Spring Boot 添加 Bucket4j。

## LOW 级别发现

### L-1: Spring Boot 版本过时

**当前**: 3.2.3（2024-02 发布）
**风险**: 可能存在未修复的 CVE，建议升级到最新的 3.2.x 或 3.3.x

### L-2: CORS allowCredentials + 宽松 Origin

Java 和 Python 服务都配置了 `allowCredentials: true`。如果生产环境 CORS origin 配置不当（如使用通配符），可导致凭证泄露。当前代码使用环境变量管理，但需确保生产环境不配置 `*`。

### L-3: 依赖版本宽松

Python 服务使用 `langgraph ^0.0.20` 和 `langchain ^0.1.0` — 这些是 0.x 版本，API 不稳定，升级可能引入 breaking changes。

## 数据分类

| 数据类型 | 分类 | 当前保护 | 不足 |

| JWT Token | 机密 | HS256 签名 | 默认密钥可预测 |
| 用户密码 | 机密 | BCrypt 哈希 | 无 |
